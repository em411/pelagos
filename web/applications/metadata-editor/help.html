Metadata Editor Help
The GRIIDC Metadata Editor is a web application that can generate ISO 19115-2 compliant metadata files in XML format. To generate a valid Metadata file, several fields in various sections have to be completed on the Metadata Editor form.
The form also contains various tabs/sections and buttons, in the forms’ toolbar.

Toolbar buttons:

Generate Metadata File:

  • This button will generate a correct but un-validated ISO 19115-2 metadata file. The file will be saved locally on the user machine in XML format.

    • Load Metadata File:

  • This button will load a previous generated file. Either from a full metadata file, a temporary saved file, or a template. The Metadata Editor expect a file with XML content.

    • Save without Validating:

  • This button will save the form a temporary metadata file, it is not fully compliant metadata file, and it is not expected to validate. This feature is useful for saving the data on the form, to continue on a later date. Or it can be used to fill out common information to the temporary file can be used as a template. The file will be saved locally on the user machine in XML format.

    • Reload the Form:

  • This button will reload a blank form.

    • Tabs/Sections:

    Dataset Contact:

  • Contains the dataset contact.

    • Dataset Information:

  • Information about the dataset.

    • Keywords:

  • This section contains the Theme, Place and Topic keywords. The add keywords to the theme and place keyword sections, simply type a keyword in the keyword field and click the add button, and simply press enter after each entry. To delete a keyword from the list, select and highlight the keyword in the list, and click delete. The topic keywords are ISO defined, and one must be selected from the Topic Keyword List. Select a topic keyword, and click on the ">>>" button to add, or double click on a keyword in the list. To delete select the keyword in the Selected Topic Keyword list and click on the "<<<" button, or double click it to remove it. When keywords are added to the Select Topic Keyword list, the keyword will disappear from the original keyword list.

    • Data Extent:

  • Data Extent Information. Preferably a bounding box coordinates are giving (longitude, latitude), together with a time period. If this is not applicable, click on the checkbox: “This dataset does not have any relevant spatial or temporal components” and fill out an appropriate description for the data.

    • Distribution Info:

  • Information about the file(s) in the dataset, size, compression technique and it’s location.
