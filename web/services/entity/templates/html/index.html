This Web service establishes a REST API for the creation and storage of Pelagos Entities.

Web Service Usage:

  • GET / - This message
  • POST /entityName - Create an entity of type entityName
  • GET /entityName/id - Retrieve an entity of type entityName identified by id
  • PUT /entityName/id - Update an entity of type entityName identified by id
  • GET /entityName - Retrieve all entities of type entityName
  • GET /entityName/validateProperty?property=value - Validate value for property of entity of type entityName
Items in italics are variable parameters.

The HTTP status code will be set appropriately to indicate success or failure:

  • Success
    • 200 OK - operation was successful
    • 201 Created - entity was created successfully
  • Client Error
    • 400 Bad Request - operation failed because invalid parameters were given
    • 401 Unauthorized - operation failed because authorization is required
    • 404 Not Found - operation failed because no entity identified by id could be found
    • 409 Conflict - operation failed because given parameters violate a uniqueness constraint
  • Server Error
    • 500 Internal Server Error - operation failed because of an unexpected condition on the server side

The response body will be JSON and include the HTTP status code, a message, and optional data, e.g.: 

{
  "code": 200,
  "message": "Great success!",
  "data": {
    "property1": "value1"
    "property2": "value2"
  }
}
